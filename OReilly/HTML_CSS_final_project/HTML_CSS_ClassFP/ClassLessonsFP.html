Lesson 6: Cascading Style Sheets: Introduction

Objective 1

Create a web page that contains at least:

  • An <h1> heading as the main heading
  • Five (short!) paragraphs
  • Five <h2> headings, one for each paragraph

Use as many other elements as you like.

Create and link to a separate CSS file that adds style to the page to:

  • Give every other paragraph a different background color.
  • Change the default font of the first <h2> heading.
  • Change the default font of the <h1> heading.

In your paragraphs, describe the style choices you made and if you used classes or ids, why you did so.

Skills applied 1

  • External CSS file
  • Background-color and font-family for <body>
  • Wrapped header and main content in <div>s for styling
  • Font-family for headings with id
  • Text and background color for <p> using class

Project 1

Thing 1 and Thing 2

Objective 2

Create styles in the document below so that:

  • The <body> of the page is in a sans-serif font.
  • Huey has a blue background and yellow text.
  • Louie has a yellow background and blue text.
  • Dewey has a different font from the rest of the page.
<!doctype html>
<html lang="en">
<head>
<title>My favorite ducks</title>
<meta charset="utf-8">
</head>
<body>
<h1>My favorite ducks are...</h1>
<ul>
<li>Huey</li>
<li>Louie</li>
<li>Dewey</li>
</ul>
</body>
</html>

Skills applied 2

  • Internal CSS
  • Font-family for <body>
  • Background and text color styling for bullet point text using ids with <li>

In addition to the lesson requirements, I wanted the color of the bullet points to remain black rather than match the color assigned to each bullet point text. In order to do so, instead of placing an id in the <li>s and modifying the color of this id in the CSS, I added <span>s with ids around the bullet point text. With the first method (using ids in the <li>s), because <li>s are block level elements, the highlighting extended across the browser window. With the second method (using ids in <span>s around the bullet point text), because <span>s are inline elements, the bullet points remained black. Also, the highlighting only extended over the text, not across the entire browser window.

Note: For Lesson 2, Objective 1, I used class in <li>s to change the background color of the text. This did not change the color of the bullet points; the bullet points remained black. Why did the bullet point not change when I used an attribute in <li>? Because in Lesson 2, Objective 1, I only changed the background color. In Lesson 6, Objective 2, I changed both the background color and the text color.

Lesson 7: Cascading Style Sheets: The Box Model

Objective

Create a web page that contains a "pull quote." A pull quote is a piece of text that is "pulled" from the main content and highlighted as a quote. You often see pull quotes in magazine or newspaper articles.

Create and link to a separate CSS file so that your HTML file contains:

  • A <body> with a light grey background color and a margin of 20px on all sides
  • A heading with an underline made from a border
  • At least three paragraphs, one of which is the pull quote
  • The pull quote should have:
    • A dark grey background color
    • White text
    • A top margin of 15px and the rest of the sides 10px
    • A stylized border (e.g., dotted or dashed)
    • Padding of 10px on all sides

Skills applied

  • External CSS file
  • Background-color and margin for <body>
  • Border styling for <h1>
  • Background and text color, margin, padding, and border styling for pull quote using <p> id

Note: Technically, I did not create a pull quote (i.e., text copied from the body text and pulled into a separate area). It would have been better to use <blockquote> for the page that I produced.

Lesson 8: Cascading Style Sheets: Inheritance

Objective

Create an external style sheet for the following HTML:

<!doctype html>
<html lang="en">
<head>
<title>My favorite ducks</title>
<meta charset="utf-8">
<link rel="stylesheet" href="duckstyle.css">
</head>
<body>
<div id="header">
<h1>My favorite ducks</h1>
<span>(A duck tale)</span>
</div>
<div id="content">
<p> I'm lucky to have many ducks in my life. I have three ducks: <span>Huey</span>,
<span>Louie</span>, and
<span>Dewey</span>.
</p>
<p> They are all very rambunctious ducks, and make a lot of noise. </p>
</div>
</body>
</html>

Your style sheet should be set up so that the duck names inherit:

  • A sans-serif font from the <body>
  • A color of light grey from the <div> with the content id
  • A text-align of center from the <p> elements

Skills applied

  • External CSS
  • Wrapped heading and paragraphs in <div>s.
  • Font-family for <body>
  • Listed fonts for Windows, Mac, cross-platform compatibility, and generic sans-serif fonts to help ensure the fonts I want are used regardless of the platform the user has.
  • Text color for <div> id
  • Text align for <p>

Note: Text color in <div> and text align in <p> could have been combined into one CSS rule for the same end result.

Lesson 9: Positioning

Objective

Using the code below, create HTML and CSS files so that:

  • Each <div> element with the box class is 100px wide and 50px high. (Use height: 50px in your CSS to set the height of a block element like <div>.)
  • The red <div> has a red background color and be positioned 50px from the top and left
  • The green <div> has a green background color and be positioned so that its left side is 10px to the right of the red <div>
  • The blue <div> has a blue background color and be positioned so that its left side is 10px to the right of the green <div>

Use the following coding:

<!doctype html>
<html lang="en">
<head>
<title>Positioning Quiz</title>
<meta charset="utf-8">
<link rel="stylesheet" href="colors.css">
</head>
<body>
<div id="container">
<div class="box" id="red">
</div>
<div class="box" id="green">
</div>
<div class="box" id="blue">
</div>
</div>
</body>
</html>

Now that you've positioned the colored <div> elements, change the CSS so that their positions are shifted over 200px from the top and 200px from the left by positioning the container <div>.

Goal: In normal document flow, these three colored boxes would appear vertically one on top of the other. Removed from normal flow, these three colored boxes appear horizontally one after the other with 10 pixels of space beween them. The following are four examples using absolute and relative positioning:

  1. Absolute positioning
  2. Relative positioning without container <div>, intentionally resulting in the same placement of the colored boxes as in absolute positioning
  3. Relative positioning using container <div>, intentionally resulting in a different placement of the colored boxes than absolute positioning
  4. Relative positioning using container <div>, intentionally resulting in the same placement of the colored boxes as in absolute positioning

Skills applied

Absolute positioning

  • Wrapped content in container <div>. Used relative positioning and set top and left position for the container <div> (relative position = content taken out of flow and placed relative to where it would normally appear in the document flow)
  • Wrapped three colored boxes in box <div>. Used absolute positioning and set the width, height, and top of each box (absolute position = content taken out of flow and placed in a specified location)
  • Wrapped each color box in a <div>. Set the background color and left position for each <div>.

Relative positioning without container <div>

(intentionally resulting in the same placement of the colored boxes as in absolute positioning)
Aimed to achieve same layout as with absolute positioning but by using relative positioning. Making boxes relative places them relative to the container element, which is also relative; the boxes are not relative to position 0,0.

  • Commented out container <div>.
  • In box <div>, changed absolute positioning to relative positioning. Changed values of left and top properties.
  • In each colored box <div>, changed values of top and left properties.

Relative positioning using container <div>

(intentionally resulting in a different placement of the colored boxes than in absolute positioning)
Explored what happened if container <div> not commented out when using relative positioning for both the box <div> and the container <div>. The result in this example is not the same as when absolute positioning was used; the boxes are further to the right than in the absolute positioning example.

  • In container <div>, kept relative positioning and top and left properties.
  • In box <div>, kept relative positioning. Changed values of left and top properties.
  • In each colored box <div>, changed values of top and left properties.

Relative positioning using container <div>

(intentionally resulting in the same placement of the colored boxes as in absolute positioning)
Aimed to achieve same layout as with absolute positioning but by using relative positioning and the container <div>. Basically, the only thing that needed to change was the value of the top of each box. (In normal flow, the boxes would have stacked on top of each other. To horizontally align them, the value of the top needed to change.)

  • In container <div>, kept relative positioning and top and left properties.
  • In box <div>, kept relative positioning. Changed values of left and top properties.
  • In each colored box <div>, changed values of top and left properties.

Lesson 10: Forms

Objective

Imagine you're a teacher and you're designing a form to test your students on a topic. (Pick whatever topic you like!)

Create a form for your test and include at least four different input types for different kinds of questions you might ask your students. Include test instructions and topic headings if you like. Add some style to your form using CSS in an external style sheet.

Skills applied

  • External CSS
  • Listed fonts for Windows, Mac, cross-platform compatibility, and generic sans-serif fonts to help ensure that the fonts I want are used regardless of the platform the user has.
  • Set font-size in <body> to medium and subsequent font sizes to percentages so that font size would change relative to each other if the user changed the text size in their browser.
  • Added background photo.
    • Reduced opacity of photo.
    • Fixed image to browser window so that image resizes with the window.
    • Centered image on the page.

  • Added background color to text boxes.
  • Four form types
    • Text boxes for short answer as well as last and first names

I used absolute/relative positioning for columns since there was no footer that would cause problems with this method; setting one column to a certain width may cause that column to overlap with a footer that spans the browser when the browser window is resized.

Lesson 11: Tables

Objective

Look in Wikipedia at your favorite topics and find an entry that has some table data.

Almost every entry has some kind of table data, but if you're struggling to find something, try the Wikipedia page for Tim O'Reilly (the founder of O'Reilly Media). You'll see on the right side of the page a box with some biography data about Tim. (Almost every person listed in Wikipedia has this kind of biography data, so pick another person if you like.)

Use the data you find in Wikipedia to create a table to hold it. Above the table, in one or more paragraphs, describe why you chose that data and your decision-making process for structuring the table to best manage that data. For instance, why did you pick rows or columns for the table headings? Did you create a separate cell for each item of data, or did you combine some pieces of data, and if so, why?

Once you've created your table, add some style to make it look nice and, most important, easy to read. Create and link to a separate CSS file.

Skills applied

  • External CSS
  • Listed fonts for Windows, Mac, cross-platform compatibility, and generic sans-serif fonts to help ensure that the fonts I want are used regardless of the platform the user has.
  • Set font-size in <body> to medium and subsequent font sizes to percentages so that the font size would change relative to each other if the user changed the text size in their browser.
  • Font-family, background-color for <body>
  • Font-size, font-weight, text-decoration, text-align for <caption>
  • Font-weight for problem class in <span>
  • Created separate styles for elements in headers (subfamily or genus names vs. species or parenthetical text under genus names).
    • Text-align, width, padding, font-weight for <th>
    • Font-weight for <th> in <tr> with subfamily or genus class
  • Created separate styles for items in <tr>.
    • Font-size for <tr> with subfamily class
    • Background-color for <tr> with species class
    • Font-weight for <span> with tabheader class in <tr>
  • Width and padding for <td>
    • Setting width in <td> prevented table from taking up entire browser window.
  • Added colspan attribute to <td> or <th> to solve validation error caused by rows having different number of columns (i.e., one column for genus names, two columns for species name and description).
  • Added cross-references within document to help user navigate page. Styled with table class.
  • Used different styling for different types of links (e.g., "back to top" link vs. source link).
    • Font-style and font-size for source id
    • Link text color for source id and table class
    • Link hover color for source id and table class
    • Link visited color for source id and table class

Lesson 13: Final Project

Objective

This lesson is your final project for the course. The goal of this project is to create an entire website. You can make this website for any subject you wish. If you are having trouble coming up with an idea, you might consider creating an online portfolio and resume. Such a website might include:

  • A main page with general information about you.
  • Links to pages with the following information about you:
    • Work experience
    • Education
    • Interests
    • Past and current websites—you can include the pages you created throughout this course.
    • Links to your favorite websites

As the final project, this website must:

  • Include elements and properties discussed in both the HTML and CSS portions of the course.
  • Use an external CSS file for all the pages on the site.
  • Observe good programming practices.
  • Validate against the W3 online validator.
  • Use good names for ids and classes.
  • Use indentation to make the code readable.
  • Be well-organized, professional, easy to navigate, and aesthetically pleasing, with no typos or syntax errors.
  • Work with:
    • IE on Windows
    • FireFox on Windows
    • Safari on Mac
    • Chrome on Mac

Be creative and have fun! You want to present yourself in a professional yet friendly way, so feel free to express yourself!
